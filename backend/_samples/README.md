# 辨識評測樣本（真實拍攝照片）

這個資料夾放**你手機實拍的卡片照片**，用來真實評測辨識準確率、挑選/調校模型。
（合成貼圖會給假高分，所以一律用真實照片。）

## 怎麼放

1. 把照片（.jpg / .png / .heic 轉 jpg）丟進這個資料夾。
2. 在 `manifest.csv` 補上每張照片對應的**正確卡號**（card_id）。

### manifest.csv 格式

```
filename,card_id
beautifly_ar_01.jpg,M2A_194/193
beautifly_ar_02.jpg,M2A_194/193
pikachu_sar_01.jpg,M2A_234/193
```

- `filename`：照片檔名（就放在本資料夾，不含路徑）。
- `card_id`：該卡正確的 id，格式 `<SET>_<num>`，例：`M2A_194/193`、`M5_114/081`。
  - 卡號含斜線是正常的（對應卡圖 `webapp/public/img/cards/<SET>/<num-with-dash>.png`）。
  - 不確定 card_id 可在 App「加入卡片」搜尋該卡，或直接在這裡寫卡名我幫你查。

## 建議涵蓋的情境（越多元越能調得準）

- **同一張卡 2~3 個角度/光線**（特別是 holo/AR 強反光，這是目前最會錯的）。
- **10~20 種不同的卡**（普卡 C/U + 亮面 AR/SAR/UR 都要）。
- 至少 1~2 張**沒有卡的空畫面**（檔名隨意，card_id 填 `NONE`）→ 驗證「不誤判」。
